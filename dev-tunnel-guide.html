Connecting Copilot Studio to a Local MCP Server
via VS Code Dev Tunnel

Copilot Insights  ·  No Azure deployment required  ·  Mac + VS Code

This guide walks through exposing the Copilot Insights MCP server running on your Mac to Microsoft Copilot Studio using a VS Code dev tunnel — no Azure Container App or cloud deployment needed.

What you need: VS Code, Python environment with project dependencies installed, a GitHub or Microsoft account (for the tunnel), and access to Copilot Studio.

Phase 1 Start the MCP Server in HTTP Mode

The MCP server supports two transport modes. By default it runs in stdio mode for Claude Code. For Copilot Studio you need to switch it to http mode, which starts a local web server on port 8000.

Open a terminal in VS Code in your project folder and run:

cd ~/GITHUB/Copilot-Insights
MCP_TRANSPORT=http python -m src.mcp_server.server

You should see uvicorn start and log:

INFO: Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
Tip: Leave this terminal running — the server must stay active for the tunnel to work. You can also add PORT=8000 to your .env file; 8000 is already the default.

Phase 2 Create a Public Dev Tunnel in VS Code

  1. Open the Ports panel in VS Code:
    • Menu: Terminal > Ports, or
    • Press Cmd+Shift+P and type Focus on Ports View
  2. Click Forward a Port and enter 8000
  3. Once the port appears in the list, right-click the row and select:
    Port Visibility > Public
  4. If prompted, sign in with your GitHub or Microsoft account
  5. Copy the Forwarded Address URL — it will look like: 
    https://abc123-8000.euw.devtunnels.ms
  6. Verify the tunnel is working by opening this in your browser: 
    https://abc123-8000.euw.devtunnels.ms/health
    You should see: {"status": "ok"}
Important: Setting visibility to Public is required. Copilot Studio runs as a cloud service and cannot reach a private tunnel. Anyone who knows the URL can query your data while the tunnel is open — close the port when you are done.

Phase 3 Connect Copilot Studio to the MCP Server

Step 1 — Enable Generative Orchestration

MCP tools require Generative Orchestration to be active on your agent.

  1. In Copilot Studio, open your agent
  2. Go to Settings > Generative AI
  3. Under Orchestration, select Generative
  4. Save the setting

Step 2 — Add the MCP Tool

  1. Go to the Tools page for your agent
  2. Select Add a tool > New tool > Model Context Protocol
  3. Fill in the MCP wizard with the following values:
FieldValue
Server nameCopilot Insights
Server descriptionAgent telemetry and governance data for Copilot Studio agents
Server URLYour tunnel URL + /mcp   e.g. https://abc123-8000.euw.devtunnels.ms/mcp
AuthenticationNone
  1. Click Connect — Copilot Studio will call your server and automatically discover all available tools
  2. Select the tools you want available in the agent (you can enable all of them) and save

Phase 4 Test It

In your agent's Test panel, try prompts like:

  • "How many production conversations happened this week?"
  • "Which connectors have the highest failure rate?"
  • "List all agents and their environments"
  • "Show me token consumption across models"

The agent will call your local server through the tunnel and return live data from your SQLite database.

Important Caveats

TopicDetail
Tunnel lifetime The tunnel is active only while VS Code is open and the port is forwarded. If you close VS Code, Copilot Studio loses the connection.
URL changes each session VS Code generates a new tunnel URL each session. You will need to update the Server URL in Copilot Studio each time you restart.
Security The server runs without an API key in local mode. Close the forwarded port in VS Code when you are done to stop public access.
Persistent URL option A free ngrok account gives you a fixed subdomain so you do not need to update the Copilot Studio config after each restart.
Available MCP tools get_summary_stats, get_conversations, get_conversation_detail, get_connector_calls, get_top_connectors, get_user_prompts, search_by_user, get_agents, get_environments, get_viva_insights, get_agent_activity, get_user_activity, run_sql
